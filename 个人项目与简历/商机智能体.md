# 商机智能体

## 问题场景

### 核心问题：并发写入导致数据覆盖丢失

假设表格现有10行数据，同一时刻两个商机消息到达：

**场景1：新增覆盖**
- 用户A发商机 → Agent读取表格（全10行）→ LLM判断：新增，应插入第11行
- 用户B发商机 → Agent读取表格（全10行，A还没写入）→ LLM判断：新增，应插入第11行
- Agent A写入第11行 ✅
- Agent B写入第11行 ❌ 覆盖A的数据
- **结果：** 用户A的商机静默丢失，无报错

**场景2：删除+更新冲突**
- 用户A删除第10行 → 表格从10行变成9行
- 用户B同时要更新第10行（但此时第10行已是原第11行数据）
- **结果：** B更新了错误的数据，A的删除意图也被扭曲

**场景3：连续删除**
- 用户A读取表格（10行）→ 要删第10行
- 用户B读取表格（10行）→ 也要删第10行
- A执行删除第10行 → 表格变成9行（原第11行变成第10行）
- B执行删除第10行 → 实际删除的是原第11行（现第10行）
- **结果：** 期望只删第10行，实际删了原第10行和原第11行两行数据

### 问题本质

- 并发写同一行，后写的直接覆盖先写的
- 没有报错，没有察觉，数据静默丢失
- 只有**新增和删除**会改变行号，引发冲突；**更新**只改内容，不影响行号，无此问题
