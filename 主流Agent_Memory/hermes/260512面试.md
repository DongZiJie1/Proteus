## Q1：Hermes 的 Hooks 系统有哪些生命周期钩子？和 OpenClaw 的差异？Pre-Compaction 前能自定义挂载吗？

**你的回答：** 根据通用经验猜了会话启动/结束、工具前后、压缩前后。差异不知道。可以挂。

**追问过程：**
- Q: Hooks 和触发层的 5 种信号源是什么关系？ → A: 没区分开，把 Hooks 和触发层信号源混在一起

**点评：** 没有精确列表，通用方向对。关键混淆：触发层的"Hooks"指外部信号源的一种，Agent 运行时的 Hooks 是事件拦截点——两者不是同一层概念。

**正确/完整答案：** Hermes 的 Hooks 按插入点分：会话周期（SessionStart→SessionEnd 及子事件）、工具执行（PreToolUse→PostToolUse→PostToolUseFailure）、LLM 调用（PreCompact→PostCompact）、Agent 生命周期。每个钩子点可注册 `command`（shell 脚本）或 `prompt`（LLM 注入提示），在 `hermes.yaml` 配置。与 OpenClaw 差异：Hermes Hooks 与自进化循环深度耦合——PreToolUseFailure 先触发 Error Classifier 分类再走 Hook；PostToolUse 达阈值后触发自进化反思。Pre-Compaction 前挂 `PreCompact` 钩子即可。

`⚠️ 方向对但细节不足`

## Q6：Memory Flush 完如果没触发 Compaction 就 /reset 了，写入的 memory 文件还在不在？能被索引到吗？

**你的回答：** 不知道。

**点评：** 考的是两条数据流的隔离性——你没区分清楚。

**正确/完整答案：** **在，能被索引到。** Memory Flush 写的是 `memory/YYYY-MM-DD.md`（磁盘文件），Compaction 写的是 `<sessionId>.jsonl`（会话日志）。两条流互不依赖——Flush 完即使 `/reset` 销毁 session，磁盘 memory 文件仍在。而且 `/reset` 的 hook 本身也会触发"会话结束自动归档"到 memory/。文件落盘后 chokidar 监听器 1.5 秒内自动重索引，下次 `memory_search` 就能搜到。关键区分：Flush/归档走"磁盘文件→索引"路径，Compaction 走"内存压缩→jsonl 追加"路径，两条线解耦。

`❌ 未掌握`

## Q7：Error Classifier 分类后的恢复路径谁执行？权限错误+用户不在线怎么处理？

**你的回答：** LLM 参与决策，系统只是把分类结果+预设建议给 LLM，LLM 在下一轮 Think 中决策。用户不在线则持久化状态等待。

**追问过程：**
- Q: Error Classifier 是自动执行还是回灌 LLM？ → A: 回灌 LLM → ✅（纠正了最初"系统自动"的说法）
- Q: 用户不在线持久化到哪里？ → A: 对话记录 → ✅（补充：也可写入 memory 文件跨 session 保留）

**点评：** 第二轮到第三轮追问纠正了认知——Error Classifier 是预处理器而非自动执行器。用户不在线的持久化路径正确。

**正确/完整答案：** Error Classifier 返回 `{type, recoverable, suggestion}` 回灌给 LLM，LLM 在下一轮 Think 决策。不自动重试。用户不在线时：①状态自然保留在 session transcript 中，续会话时恢复；②可写入 memory 文件（通过自进化循环或 Flush），跨 session 可检索。

`✅ 基本正确`

## Q8：Context Engine 预检索用什么字段查询？结果怎么拼进上下文？对话中未 Flush 的内容能命中吗？

**你的回答：** 用当前用户输入做 RAG 匹配。结果拼在用户最新消息之前。未 Flush 不能命中。

**追问过程：**
- Q: 拼在用户消息区还是系统提示区？ → A: 修正为"用户最新消息之前" → ⚠️ 位置对但表述混淆
- Q: 没 Flush 的内容能命中吗？ → A: 不能 → ✅

**点评：** 命中逻辑对，拼装位置对但表述中把"独立片段"说成"作为用户消息"——预检索片段是独立注入块，不带有 user 角色。

**正确/完整答案：** 预检索用当前用户消息作为查询。结果拼在第 5 步位置（对话历史切片之后、当前用户消息之前），是独立的上下文片段。依赖 Retrieval Index，只索引 md 文件——对话中未 Flush 到 memory/ 的内容无法被预检索命中。这就是 Hermes 要有 Memory Flush + 自进化循环的原因。

`⚠️ 方向对但表述需更精确`

## Q11：Heartbeat 和 Cron 的区别？heartbeat 间隔？上一循环未结束怎么办？

**你的回答：** 不知道。

**正确/完整答案：** Heartbeat 是轻量周期性唤醒（轮询驱动），让 Agent 自检"有无待办/消息积压/HEARTBEAT.md 任务？"，间隔通常 30-60 秒。Cron 是定时任务（时间驱动），到点执行具体任务（如"每天 9 点发简报"）。区别：Heartbeat 不绑定具体任务，Cron 绑定具体任务。如果上一循环未结束：Agent loop 单线程串行，新 heartbeat 事件排队等待，不会并发抢占——Gateway 放入 session 队列等待当前 loop 结束。

`❌ 未掌握`

**你的回答：** 不知道。

## Q14：LLM 连续 3 轮返回同样工具调用，Hermes 有循环检测吗？和 Error Classifier 什么关系？

**你的回答：** 不知道。

**正确/完整答案：** 材料没有显式独立的"循环检测器"模块。防重复核心靠 **Error Classifier**——捕获失败后直接归类+附建议，LLM 拿到分类结果就不会再"原样重试"。没有 Error Classifier 时，弱模型会连续调用同一失败命令 5 次（见 260506 Q6 举例）。Error Classifier 就是事实上的循环打断机制：把"试错循环"变成"识别→走恢复路径"。

`❌ 未掌握`
